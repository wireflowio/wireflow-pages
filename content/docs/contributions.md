# Contributions
We welcome contributions from the community to help improve Wireflow! Whether it's reporting bugs, suggesting new features, or submitting code changes, your input is valuable to us. Here are some ways you can contribute:
## Reporting Issues
If you encounter any bugs or issues while using Wireflow, please report them on our [GitHub Issues](https://github.com/wireflowio/wireflow/issues) page. Be sure to include as much detail as possible, including steps to reproduce the issue, your environment, and any relevant logs or screenshots.
## Suggesting Features
We are always looking for ways to enhance Wireflow. If you have ideas for new features or improvements, please share them on our [GitHub Discussions](https://github.com/wireflowio/wireflow/discussions) page.

## Submitting Code Changes
If you would like to contribute code to Wireflow, first choose a repo, if you want choose wireflow, please follow these steps:
1. Fork the Wireflow repository on GitHub.
2. Clone your forked repository to your local machine.
   ```bash
    git clone https://github.com/wireflowio/wireflow.git
    cd wireflow
    ```
3. Create a new branch for your changes.
4. Make your changes and commit them with a descriptive message.
   ```bash
   git add .
   git commit -m "Your descriptive commit message"
   ```
5. Push your changes to your forked repository.
   ```bash
   git push origin your-branch-name
   ```
6. Create a pull request on the Wireflow repository.
7. Wait for feedback and be prepared to make changes based on reviewer suggestions.
## Code of Conduct
We expect all contributors to adhere to our [Code of Conduct](https://github.com/wireflow/wireflow/blob/main/CODE_OF_CONDUCT.md). Please be respectful and considerate in all interactions within the Wireflow community.
Thank you for your interest in contributing to Wireflow! We look forward to collaborating with you to make Wireflow even better.
